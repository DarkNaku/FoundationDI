# go 명령어 - RED 단계 시작

plan.md에서 다음 미표시 테스트를 찾아 RED 단계를 구현합니다.

## 실행 과정

1. **plan.md 분석**: `[ ]`로 표시된 첫 번째 미표시 테스트 찾기
2. **테스트 작성**: 실패하는 최소한의 테스트 코드 작성
3. **테스트 실행**: 예상대로 실패하는지 확인
4. **결과 표시**: 테스트 코드와 실패 메시지 표시

## 특징

✅ **Red 단계 규칙**:
- 테스트는 반드시 실패해야 함
- 프로덕션 코드는 수정하지 않음
- 테스트 코드만 작성
- 테스트 이름은 한국어 "should~" 형식

✅ **Kent Beck의 TDD 원칙 준수**:
- Red → Green → Refactor 사이클
- 한 번에 하나의 테스트만 작성
- 가장 간단한 실패 케이스부터 시작

## 사용 예

```
go
```

완료되면 다음 단계: `/green` 또는 `/refactor`
