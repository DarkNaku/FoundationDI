# green 명령어 - GREEN 단계 실행

현재 실패하는 테스트를 통과시키는 최소 코드를 작성합니다.

## 실행 내용

1. **최소 코드 구현**: 테스트를 통과시키는 가장 간단한 코드만 작성
2. **모든 테스트 실행**: 기존 테스트도 계속 통과하는지 확인
3. **결과 보고**: 통과한 테스트 목록 표시

## GREEN 단계 규칙

✅ **반드시 지켜야 할 규칙**:
- 테스트는 통과해야 함 (✅)
- 가장 간단한 구현만 함
- 중복 코드는 무시 (Refactor 단계에서 처리)
- 리팩토링하지 말 것
- 기존 기능은 유지

✅ **최소 구현의 예**:
- 하드코딩도 괜찮음
- 모든 경우를 처리할 필요 없음 (테스트에서 요구하는 것만)
- 단순함이 최고의 미덕

## 사용 예

```
/green
```

다음 단계: `/refactor` (코드 정리) 또는 `/go` (다음 테스트)
